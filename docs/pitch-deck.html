AI fluency

AI is the engine, not a wrapper.

👁 Vision review

The model reviews every page's real screenshot like a senior designer — CTA contrast, hierarchy, trust signals — findings become issues with the same 1:1 rigor as automated checks.

🔍 Markup review

Reads real extracted DOM data and catches what rules can't: weak titles, mismatched schema, suspicious link text — grounded, never invented.

🛠 Fix generator

Writes the fix card for each issue using the actual offending element's HTML — page-specific code your developer pastes in, not generic advice.

💬 Grounded chat

Answers strictly from the stored analysis — real scores, real issue names — and refuses out-of-scope questions. Provider-agnostic: Groq, Gemini, any OpenAI-compatible endpoint, zod-validated JSON everywhere.

Website Doctor04 / 07
Why it's different

The honest scanner.

  • 1:1 invariant, DB-enforced — every failed check = exactly one issue, same severity; counts reconcile in every view
  • No silent failures — a check that couldn't run shows orange "Not run — error" with the real reason. Never a fake pass, never a fake 0/100
  • One-click re-run — any check or analyzer, offline from stored artifacts where possible
  • Resolved marks survive re-runs — the tool respects your triage

Trust is the product. If users can't trust the numbers, an audit tool is decoration.

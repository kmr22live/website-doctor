Website Doctor03 / 10
How it works

From URL to fixed website — six steps.

1Paste a URL

One input. No setup, no snippet to install, no account gymnastics.

2Real crawl

Headless Chromium visits up to 10 pages — real navigation, screenshots, full HTML, response headers. Live progress streams to the screen.

3Deep audits

60+ rule checks + Lighthouse (LCP/CLS/TBT) + axe-core accessibility + security headers, TLS cert, broken links — per page.

4AI review

The model reviews each screenshot like a senior designer, audits the markup, checks cross-page consistency, and writes an executive summary.

5Issues → fixes

Every failed check becomes one prioritized issue: impact, paste-in code, and the exact broken elements captured from the page.

6Act & verify

Mark resolved, re-run single checks to confirm fixes, ask the AI assistant anything about the findings, export the PDF for the client.

crawl → audit → AI → issues → fixes → verify — a full quality loop, not a one-shot report.

Why it's different

The honest scanner.

  • 1:1 invariant, DB-enforced — failed checks = issues, same severity, counts reconcile in every view
  • No silent failures — "Not run — error" rows with the real reason. Never a fake pass, never a fake 0/100
  • One-click re-run — per check or per analyzer, offline from stored artifacts where possible
  • Resolved marks survive re-runs — the tool respects your triage

Trust is the product. If users can't trust the numbers, an audit tool is decoration.

Demo limits — config, not code

Capped on free tiers. Uncapped in one env change.

  • 100% free infrastructure — free-tier server + Groq free AI. Demo caps exist only to respect their rate limits: vision on 2 pages, Lighthouse on 5, 3s between AI calls
  • The engine has no limits — clone, pnpm dev, and the same pipeline audits all 10 pages at full depth: every analyzer, every AI review
  • Rate-limited ≠ hidden — a throttled stage shows an honest "Not run — error" row with the real reason and a one-click re-run. Results are never faked

We treat rate limits like failed checks: transparently. A QC tool that fakes its own results can't be trusted to QC anything.
